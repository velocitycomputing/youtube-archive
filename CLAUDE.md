# CLAUDE.md — youtube-archive

This repo holds the user's personal YouTube watch-history archive: one Markdown file per video, with frontmatter, a 2-paragraph summary, and the transcript.

## When working in this repo

- **DO NOT** invent video entries by hand. The archive is auto-generated by `ai-clients/scripts/youtube_archive_write.py`.
- **DO NOT** edit transcript text — it's verbatim from YouTube.
- **OK to edit:** frontmatter `status`, `routed_to`, `proposed_tags`, `proposed_entities` as the user triages.
- **OK to edit:** the `## Summary` body if the user wants to refine an auto-generated summary.
- Branch protection on `main`: PR required (per PLAN.md §14.99.I in the parent `current-focus` umbrella).

## File-naming

`videos/YYYY-MM-DD-<video-id>-<slug>.md` where:
- `YYYY-MM-DD` = `watched_date` (most-recent occurrence in history).
- `<video-id>` = the 11-char YouTube video ID.
- `<slug>` = first 30 chars of the title, lowercased, alphanumerics + `-`.

The `<video-id>` is the source of truth for identity. If a video is re-watched and the `watched_date` changes, the filename is rewritten (old file removed, new file created — `git mv` if the title hash is unchanged).

## index.md

Regenerated on every ingest run. Lists videos newest-first with title / channel / watched_date / proposed_tags / status. Helpful for grep'ing without the dashboard.

## Triage states

`status:` field in frontmatter, closed-vocabulary:
- `new` — auto-ingested; awaiting user review.
- `reviewed` — user has eyeballed it; no further action chosen.
- `routed-to-health` — user sent the summary into the Health function set (e.g., as a `research/inbox/` entry).
- `routed-to-wealth` / `routed-to-research` / etc. — analogous for other function sets.
- `archived` — user marked irrelevant; hide from default dashboard list.
- `rejected` — user actively don't-want; keep the file but exclude from the dashboard entirely.
- `blocked` — matched a denylist rule (channel / title-keyword / tag) at ingest or write time. File is preserved for reversibility — flip status back to `new` to restore. See `ai-clients/config/youtube-denylist.yaml` (PLAN.md §14.99.W.3).

## Pipeline reference

Full pipeline lives in `velocitycomputing/ai-clients`:
- W.A `scripts/youtube_takeout_ingest.py` — Takeout export → normalized watch list.
- W.B `scripts/youtube_transcript_fetch.py` — filter ≥10 min + fetch transcripts.
- W.C `scripts/youtube_summarize.py` — summarize (local Qwen3-14B → haiku fallback) + tag (haiku against TAGS.md + ENTITIES.md).
- W.D `scripts/youtube_archive_write.py` — write per-video markdown files into THIS repo + regenerate index.md.
- W.E ai-clients-dashboard `/watched-youtube` surface — read the archive, action buttons.
- W.F `config/scheduled-jobs.yaml` job `youtube-history-ingest` — daily 08:00 MT cron.
