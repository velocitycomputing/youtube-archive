# CLAUDE.md ‚Äî youtube-archive

This repo holds the user's personal YouTube watch-history archive: one Markdown file per video, with frontmatter, a 2-paragraph summary, and the transcript.

## When working in this repo

- **DO NOT** invent video entries by hand. The archive is auto-generated by `ai-clients/scripts/youtube_archive_write.py`.
- **DO NOT** edit transcript text ‚Äî it's verbatim from YouTube.
- **OK to edit:** frontmatter `status`, `routed_to`, `proposed_tags`, `proposed_entities` as the user triages.
- **OK to edit:** the `## Summary` body if the user wants to refine an auto-generated summary.
- Branch protection on `main`: PR required (per PLAN.md ¬ß14.99.I in the parent `current-focus` umbrella).

## File-naming

`videos/YYYY-MM-DD-<video-id>-<slug>.md` where:
- `YYYY-MM-DD` = `watched_date` (most-recent occurrence in history).
- `<video-id>` = the 11-char YouTube video ID.
- `<slug>` = first 30 chars of the title, lowercased, alphanumerics + `-`.

The `<video-id>` is the source of truth for identity. If a video is re-watched and the `watched_date` changes, the filename is rewritten (old file removed, new file created ‚Äî `git mv` if the title hash is unchanged).

## index.md

Regenerated on every ingest run. Lists videos newest-first with title / channel / watched_date / proposed_tags / status. Helpful for grep'ing without the dashboard.

## Triage states

`status:` field in frontmatter, closed-vocabulary:
- `new` ‚Äî auto-ingested; awaiting user review.
- `reviewed` ‚Äî user has eyeballed it; no further action chosen.
- `routed-to-health` ‚Äî user sent the summary into the Health function set (e.g., as a `research/inbox/` entry).
- `routed-to-wealth` / `routed-to-research` / etc. ‚Äî analogous for other function sets.
- `archived` ‚Äî user marked irrelevant; hide from default dashboard list.
- `rejected` ‚Äî user actively don't-want; keep the file but exclude from the dashboard entirely.

## Pipeline reference

Full pipeline lives in `velocitycomputing/ai-clients`:
- W.A `scripts/youtube_takeout_ingest.py` ‚Äî Takeout export ‚Üí normalized watch list.
- W.B `scripts/youtube_transcript_fetch.py` ‚Äî filter ‚â•10 min + fetch transcripts.
- W.C `scripts/youtube_summarize.py` ‚Äî summarize (local Qwen3-14B ‚Üí haiku fallback) + tag (haiku against TAGS.md + ENTITIES.md).
- W.D `scripts/youtube_archive_write.py` ‚Äî write per-video markdown files into THIS repo + regenerate index.md.
- W.E ai-clients-dashboard `/watched-youtube` surface ‚Äî read the archive, action buttons.
- W.F `config/scheduled-jobs.yaml` job `youtube-history-ingest` ‚Äî daily 08:00 MT cron.
